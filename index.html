Problem

Probabilistic inference is not deterministic execution authorization.

When upstream sources conflict — sensors disagree, models hallucinate, channels go Byzantine — probabilistic guardrails average conflicting signals into a single output. That output can trigger unsafe execution. Content filters catch bad outputs. Nobody gates the actions.

Solution

MGOS enforces a hard boundary between inference and execution.

The kernel sits at the last point before actuation. It does not modify actions — it authorizes or blocks them. Contradiction reduces available actions to zero: BLOCK, plus a hash-verified receipt explaining why. MGOS blocks execution on conflict — never averages.

Formal Properties
// TERM

Termination

No unbounded loops, no recursion, no data-dependent iteration. Decision completes in bounded time by construction. Testable: instrument wall-clock; verify ceiling holds.

// SAFE

Fail-Safe

Contradiction or non-majority across input channels triggers BLOCK — never pass-through, never averaged. Testable: inject conflicting frames; verify BLOCK on every conflict.

// DET

Determinism

Identical inputs always produce identical outputs under pinned environment. Bitwise reproducible. Testable: replay input sequence; compare output hashes.

// AUD

Audit Receipts

Every decision emits a hash-verified, tamper-evident receipt. Chain-linked witnesses. Replayable by third parties. Testable: verify receipt hash against decision log.

Integration

Minimal contract for integration partners.

MGOS sits between your inference stack and your execution layer. It receives a decision frame, returns ALLOW or BLOCK, and emits a receipt.

// IO Contract
Input: decision_frame { channels[], policy_ref, timestamp }
Output: { decision: ALLOW | BLOCK, state_id, flags, receipt_hash }
Where: Layer 4 — between inference (L0–L3) and execution (L5)
01 AI inference produces a proposed action with channel evidence
↓
02 MGOS evaluates channels against policy — TMR voting, deterministic selection
↓
03a Majority agreement → ALLOW + receipt → actuator executes
03b Conflict / no majority → BLOCK + receipt → safe stop

Failure Modes

contradiction → BLOCK + conflict_detected flag
no_majority → BLOCK + insufficient_consensus flag
timeout → BLOCK + bounded_time_exceeded flag
